## 슬라이드 3: 인증 시스템 메커니즘
- Spring Security 기반 로그인 프로세스
  - UserController가 로그인 페이지 제공
  - Spring Security가 인증 처리
  - PrincipalDetailService가 사용자 정보 로드
  - PrincipalDetail이 UserDetails 구현
- BCrypt 알고리즘으로 비밀번호 암호화
  - 단방향 해시 함수로 원본 복구 불가능
  - 솔트(salt) 자동 적용으로 레인보우 테이블 공격 방지
- 세션 기반 사용자 상태 관리
  - SecurityContextHolder에 인증 정보 저장
  - 최대 1개 세션 제한으로 동시 로그인 방지
- 자동 로그인(Remember Me) 기능
- 다중 인증 옵션 지원

## 슬라이드 4: 사용자 프로필 관리 메커니즘
- 개인 정보 관리 프로세스
  - UserController의 userInfo 메소드로 정보 조회
  - SecurityContextHolder에서 현재 사용자 정보 획득
  - UserService를 통한 데이터 처리
- 프로필 사진 업로드 및 크롭 기능
- 환경 설정 관리 시스템
  - UserDTO와 UserEntity 간 변환 로직
  - 설정 변경 시 즉시 DB 반영
- 개인정보 보호 제어
- 계정 연결 옵션

---
## 슬라이드 5: 권한 부여 프레임워크 메커니즘
- 역할 기반 접근 제어(RBAC)
  - UserEntity의 role 필드로 권한 관리
  - "ROLE_USER", "ROLE_ADMIN" 등 역할 구분
  - CustomSecurityConfig에서 URL별 접근 권한 설정
- 권한 관리 시스템
  - GrantedAuthority 인터페이스 구현으로 권한 표현
  - PrincipalDetail의 getAuthorities() 메소드로 권한 제공
- 사용자 권한 수준
  - SecurityFilterChain에서 URL 패턴별 접근 제어
  - hasRole() 메소드로 특정 역할 요구
- 콘텐츠 접근 제한
- 보안 정책 시행

## 슬라이드 6: 비밀번호 관리 메커니즘
- 안전한 비밀번호 재설정 워크플로우

- 비밀번호 이력 추적
  - 이전 비밀번호 재사용 방지
  - 주기적 변경 유도

- 보안 질문 복구

## 슬라이드 10: 백엔드 구현 메커니즘
- 사용자 엔티티 구조
  - JPA Entity 클래스로 DB 테이블 매핑
  - BaseEntity 상속으로 감사 필드(생성일, 수정일) 자동화
  - 관계 매핑으로 다른 엔티티와 연결
- 인증 서비스
  - Spring Security 프레임워크 활용
  - UserDetailsService 구현으로 사용자 정보 로드
  - AuthenticationManager로 인증 처리
- 프로필 서비스
  - UserService 인터페이스 정의
  - UserServiceImpl에서 비즈니스 로직 구현
  - DTO-Entity 변환 로직
- 보안 구성
  - CustomSecurityConfig 클래스에서 보안 규칙 정의
  - SecurityFilterChain 구성
  - 정적 리소스 보안 예외 처리
- 데이터 유효성 검사 및 정제
  - Bean Validation으로 입력 검증
  - XSS 방지 필터링
  - SQL 인젝션 방지

## 슬라이드 11: 데이터베이스 스키마 메커니즘
- 사용자 테이블 구조
  - id(PK), username, password, email 등 필수 필드
  - role 필드로 권한 관리
  - point 필드로 포인트 시스템 구현
- 프로필 정보 저장
  - 사용자 테이블에 통합 저장
  - 확장 가능한 구조 설계
- 인증 기록
  - 로그인 이력 별도 테이블 관리
  - IP, 기기 정보 등 보안 데이터 저장
- 세션 관리
  - Spring Session 활용 가능
  - DB 기반 세션 저장소 구현
- 다른 엔티티와의 관계
  - 주문, 리뷰, 레시피 등과 @OneToMany 관계
  - 외래 키 제약조건으로 데이터 무결성 유지
